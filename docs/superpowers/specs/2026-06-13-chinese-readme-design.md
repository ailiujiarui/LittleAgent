# 中文 README 设计

## 目标

新增 `README.zh-CN.md`，为中文开发者完整介绍 LittleAgent 的定位、
核心优势、主要能力、部署方式与安全边界；同时在英文 `README.md`
顶部增加中文文档入口。

## 受众

- 希望快速部署 QQ AI 机器人的个人开发者。
- 希望基于插件、工具调用或 MCP 扩展 Agent 的 Python 开发者。
- 需要理解项目架构、安全边界和当前能力范围的贡献者。

## 内容结构

1. 项目名称、中文定位和语言导航。
2. 核心优势：
   - 面向 QQ 场景的 OneBot v11 直连接入。
   - 小型、清晰、可测试的 Python 架构。
   - 同会话顺序处理、不同会话并发处理。
   - OpenAI 兼容模型与结构化工具调用。
   - 内置插件系统、事件订阅和插件 KV 存储。
   - 普通群消息静默归档、按需读取，不因监听群聊而自动刷屏。
   - Markdown 与 SQLite 组合的可检查记忆。
   - stdio MCP 工具导入、主动推送、Drift 与 Dashboard 扩展能力。
3. 能力总览表，区分已接入运行时的能力与可复用模块。
4. 简化架构和消息处理流程。
5. 环境要求、安装、配置、初始化和启动命令。
6. NapCat 反向 WebSocket 配置与 QQ 私聊、群聊验证方式。
7. 群消息读取插件的行为、限制和调用示例。
8. 插件目录结构与最小插件示例。
9. 安全实践、测试命令、项目结构和当前边界。

## 准确性边界

- 只描述当前仓库已实现并有代码或测试支撑的能力。
- 明确群消息归档只覆盖机器人在线期间收到的已配置群消息。
- 明确普通群消息在 `require_at = true` 时不会触发 Agent 回复。
- 不把尚未在 `AppRuntime` 中自动调度的主动推送或 Drift 模块描述为
  开箱即用的后台任务。
- 不承诺生产级高可用、完整权限系统或跨实例分布式能力。

## 安全要求

- 所有 API key 均使用 `${DEEPSEEK_API_KEY}` 等环境变量占位符。
- QQ 号、群号、端口 token 使用虚拟值或文字占位符。
- 不引用本地 `config.toml`、NapCat 二维码、QQ 登录数据、日志或数据库。
- 提醒用户不要提交 `config.toml`、`workspace/` 和 NapCat 运行态目录。
